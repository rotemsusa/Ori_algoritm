מהו אלגוריתם?

אלגוריתם הוא רצף הוראות ברור ומסודר שמוביל לפתרון של בעיה. כל שלב חייב להיות מובן וחד-משמעי, והאלגוריתם חייב להסתיים אחרי מספר סופי של שלבים.

🍰 דוגמה מהחיים

מתכון לעוגה הוא אלגוריתם: יש סדר פעולות מדויק (קלוט ביצים → ערבב → אפה 30 דקות), וכל מי שיעקוב אחריו יקבל את אותה תוצאה.

3 תכונות של אלגוריתם טוב 1. ברור — כל שלב מובן ולא משתמע לשתי פנים
2. מסודר — השלבים בסדר הנכון
3. מסתיים — מגיע לסוף ולא נתקע לנצח
💡 טיפ למבחן: אלגוריתם הוא לא שפת תכנות ולא תוכנית מחשב. אפשר לכתוב אותו במילים, בציור (תרשים זרימה) או בקוד.

✍️ איך כותבים אלגוריתם — שלב אחר שלב

נניח שצריך לפתור: "קלוט שני מספרים והדפס את הגדול מביניהם". כותבים את האלגוריתם קודם במילים, ורק אז מתרגמים לקוד:

שלב 1 — האלגוריתם במילים:

  1. קלוט מספר ראשון ושמור אותו במשתנה a
  2. קלוט מספר שני ושמור אותו במשתנה b
  3. אם a גדול מ-b → הדפס את a
  4. אחרת → הדפס את b
⬇ מתרגמים לקוד ⬇

שלב 2 — אותו אלגוריתם בקוד:

a = int(input("מספר ראשון: ")) b = int(input("מספר שני: ")) if a > b: print(a) else: print(b)
דוגמה שנייה: אלגוריתם לציור ריבוע (במילים)
  1. הכן את הצב הגרפי
  2. חזור 4 פעמים על: נוע 100 צעדים קדימה, ואז הסתובב 90 מעלות
  3. סיים — התקבל ריבוע
💡 הסוד: תמיד כדאי לכתוב את האלגוריתם קודם במילים שלך, ואז להפוך כל שלב לשורת קוד. ככה לא מתבלבלים.
↑ חזרה למעלה

הצב הגרפי (turtle)

הצב הגרפי הוא כלי בפייתון שמצייר על המסך לפי פקודות. מדמיינים צב קטן שמחזיק עט — הוא נע, מסתובב, ומשאיר קו אחריו.

🛠️ הפקודות החשובות

פקודהמה היא עושה
t.forward(100)נע 100 צעדים קדימה (ומצייר קו)
t.backward(50)נע 50 צעדים אחורה
t.right(90)מסתובב 90 מעלות ימינה
t.left(45)מסתובב 45 מעלות שמאלה
t.penup()מרים את העט (נע בלי לצייר)
t.pendown()מוריד את העט (חוזר לצייר)

⬜ ציור ריבוע

import turtle t = turtle.Turtle() for i in range(4): # חוזרים 4 פעמים t.forward(100) # צלע t.right(90) # פינה

🔍 הסבר הקוד — שורה אחר שורה

import turtle
מביא את הכלי של הצב הגרפי לתוך התוכנית.
t = turtle.Turtle()
יוצר צב חדש ושומר אותו במשתנה t.
for i in range(4):
לולאה שחוזרת 4 פעמים על השורות שמתחת (אלו עם הרווח).
t.forward(100)
מצייר צלע אחת באורך 100.
t.right(90)
מסתובב 90° לפינה הבאה. אחרי 4 חזרות → ריבוע סגור.
💡 הנוסחה הקסומה: זווית הסיבוב = 360 חלקי מספר הצלעות.
משולש → 360÷3 = 120° · ריבוע → 360÷4 = 90° · מחומש → 360÷5 = 72° · משושה → 360÷6 = 60°

✂️ ציור שני קווים נפרדים

t.forward(100) # קו ראשון t.penup() # מרים עט t.forward(50) # זז בלי לצייר t.pendown() # מוריד עט t.forward(100) # קו שני
↑ חזרה למעלה

משתנים וסוגיהם

משתנה הוא "קופסה" עם שם, ששומרת בתוכה ערך. מגדירים אותו עם סימן = (השמה): השם משמאל, הערך מימין.

age = 13 # השם age מקבל את הערך 13 name = "Rotem" # השם name מקבל מחרוזת

🔤 ארבעת סוגי המשתנים

סוגמה זהדוגמה
intמספר שלםx = 7
floatמספר עשרוני (עם נקודה)x = 3.5
strמחרוזת / טקסט (במרכאות)x = "hi"
boolאמת או שקרx = True
⚠️ מלכודת נפוצה: כשיש מרכאות — זו תמיד מחרוזת (str), גם אם בפנים יש מספרים!
x = 5 זה int · x = "5" זה str
תרגול זיהוי a = 100 → int (מספר שלם)
b = 99.9 → float (יש נקודה)
c = "100" → str (יש מרכאות)
d = False → bool (אמת/שקר)
↑ חזרה למעלה

הפעולה type()

הפעולה type() אומרת לנו מה הסוג של ערך או משתנה. שמים בתוך הסוגריים את מה שרוצים לבדוק.

print(type(5)) # <class 'int'> print(type(3.5)) # <class 'float'> print(type("hi")) # <class 'str'> print(type(True)) # <class 'bool'>

🧠 דוגמה שדורשת מחשבה

x = "7" print(type(x)) # <class 'str'>

למרות שיש בפנים את הספרה 7 — היא בתוך מרכאות, ולכן הסוג הוא str ולא int.

💡 שימי לב: type() בודק קודם את התוצאה. לדוגמה type(4 + 2) יחזיר int, כי 4+2=6 שהוא מספר שלם.
↑ חזרה למעלה
5

קלט ופלט (input / print)

📤 פלט — print()

הפקודה print() מדפיסה למסך. מה שבתוך מרכאות מודפס כטקסט בדיוק כמו שהוא; מה שבלי מרכאות — מחושב.

print("שלום") # שלום print(3 + 4) # 7 (חישוב) print("3 + 4") # 3 + 4 (טקסט) print("גיל:", 13) # גיל: 13 (פסיק מפריד)

📥 קלט — input()

הפקודה input() קולטת מה שהמשתמש מקליד. חשוב מאוד: input() תמיד מחזיר מחרוזת (str)!

רוצים לקלוטאיך כותבים
טקסט (שם, עיר)x = input("שם: ")
מספר שלםx = int(input("גיל: "))
מספר עשרוניx = float(input("גובה: "))
⚠️ המלכודת הכי חשובה במבחן!
אם קולטים מספר בלי int(), הוא נשאר מחרוזת — וחיבור מחרוזות פשוט מצמיד אותן:
n = input("מספר: ") # המשתמש הקליד 4 print(n + n) # 44 (ולא 8!)
עם int() זה היה מחשב נכון: 4+4=8.

🧩 דוגמה מלאה: קלט → חישוב → פלט

a = int(input("מספר ראשון: ")) b = int(input("מספר שני: ")) print("הסכום הוא", a + b)

🔍 הסבר הקוד — שורה אחר שורה

ככה "קוראים" קוד: עוברים שורה-שורה ומבינים מה כל אחת עושה. זה בדיוק מה שמבקשים בשאלות "מה התוכנית עושה?".

a = int(input("מספר ראשון: "))
מבקש מהמשתמש מספר, הופך אותו למספר שלם עם int(), ושומר במשתנה a.
b = int(input("מספר שני: "))
אותו דבר — קולט מספר שני ושומר במשתנה b.
print("הסכום הוא", a + b)
מחשב a + b ומדפיס את התוצאה אחרי הטקסט "הסכום הוא".
דוגמה לפירוק קוד עם מלכודת — מה יודפס?
x = "10"
x הוא מחרוזת (יש מרכאות!), לא מספר.
print(x * 3)
מחרוזת כפול 3 = שכפול → מודפס 101010 (ולא 30).
↑ חזרה למעלה

🔢 סדר פעולות חשבון

פייתון מחשב לפי הסדר: סוגריים ← חזקה ← כפל/חילוק/שארית ← חיבור/חיסור.

דוגמאות מודרגות 3 + 4 * 2 → קודם 4×2=8, אז 8+3 = 11
(3 + 4) * 2 → קודם הסוגריים 7, אז 7×2 = 14
10 - 2 ** 3 → קודם חזקה 8, אז 10−8 = 2
⚠️ ההבדל בין / לבין //
/ תמיד מחזיר float (גם 10/2 נותן 5.0)
// מחזיר חילוק שלם בלי שארית: 17 // 5 = 3
↑ חזרה למעלה
7

שארית החלוקה (%)

האופרטור % (מודולו) מחזיר את השארית שנשארת אחרי חלוקה.

איך מחשבים — צעד אחר צעד 17 % 5 → כמה פעמים 5 נכנס ב-17? שלוש פעמים (5×3=15). מה נשאר? 17−15 = 2

20 % 4 → 4 נכנס ב-20 חמש פעמים בדיוק (4×5=20), לא נשאר כלום → 0
⚠️ מקרה מבלבל: כשהמספר הראשון קטן מהשני, הוא נכנס 0 פעמים, והשארית היא המספר עצמו:
7 % 10 = 7 · 3 % 8 = 3
💡 השימוש הכי נפוץ: זוגי או אי-זוגי?
אם n % 2 == 0 → המספר זוגי (מתחלק ב-2 בלי שארית)
אם n % 2 == 1 → המספר אי-זוגי
↑ חזרה למעלה

אופרטורי יחס

אופרטורי יחס משווים בין שני ערכים, והתוצאה היא תמיד True או False (כן או לא).

אופרטורמשמעותדוגמהתוצאה
==שווה ל-4 == 4True
!=שונה מ-4 != 5True
<קטן מ-6 < 4False
>גדול מ-6 > 4True
<=קטן או שווה3 <= 4True
>=גדול או שווה3 >= 4False
⚠️ ההבדל הכי חשוב: = מול ==
= (אחד) זה השמה — שם ערך בתוך משתנה: x = 5
== (כפול) זה בדיקה — שואל האם שווה: x == 5
תרגול מהיר — True או False? 10 == 10 → True (שווים)
7 != 7 → False (הם כן שווים, אז לא שונים)
5 >= 5 → True (שווה נחשב)
2 > 9 → False (2 לא גדול מ-9)
↑ חזרה למעלה
